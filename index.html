Setup Guide

Follow these steps once to get your BCA app live on the web and connected to a database for all your athletes.

BEFORE YOU START

You need: a GitHub account (free) and a Supabase account (free at supabase.com). The whole setup takes about 20 minutes.

PART 1 — SUPABASE DATABASE
1

Create Supabase Account

Go to supabase.com → Sign up free → Create a new project. Name it bca-tracker. Save your database password somewhere safe.

2

Run the Database Schema

In your Supabase project, go to SQL Editor → New Query. Paste and run this:

create table athletes ( id uuid default gen_random_uuid() primary key, nickname text unique not null, pin text not null, age int not null, age_group text not null, equip text not null, phase text not null, created_at timestamptz default now() ); create table workout_logs ( id uuid default gen_random_uuid() primary key, athlete_id uuid references athletes(id), nickname text, age_group text, phase text, day text, session text, exercises jsonb, notes text, equipment text, created_at timestamptz default now() ); create table ruck_logs ( id uuid default gen_random_uuid() primary key, athlete_id uuid references athletes(id), nickname text, age_group text, phase text, ruck_date date, duration int, distance numeric, pack_weight int, elevation int, avg_hr int, rpe int, notes text, created_at timestamptz default now() );
3

Get Your API Keys

In Supabase go to Settings → API. Copy your Project URL and your anon/public key. You'll need these in Part 2.

4

Enable Security Policies

In SQL Editor → New Query, paste and run:

ALTER TABLE athletes ENABLE ROW LEVEL SECURITY; ALTER TABLE workout_logs ENABLE ROW LEVEL SECURITY; ALTER TABLE ruck_logs ENABLE ROW LEVEL SECURITY; CREATE POLICY "anon_read" ON athletes FOR SELECT TO anon USING (true); CREATE POLICY "anon_insert" ON athletes FOR INSERT TO anon WITH CHECK (true); CREATE POLICY "anon_update" ON athletes FOR UPDATE TO anon USING (true); CREATE POLICY "anon_read" ON workout_logs FOR SELECT TO anon USING (true); CREATE POLICY "anon_insert" ON workout_logs FOR INSERT TO anon WITH CHECK (true); CREATE POLICY "anon_read" ON ruck_logs FOR SELECT TO anon USING (true); CREATE POLICY "anon_insert" ON ruck_logs FOR INSERT TO anon WITH CHECK (true);

This blocks all deletes and prevents anyone from editing logged sessions.

PART 2 — DEPLOY TO GITHUB PAGES
5

Create a GitHub Repository

Go to github.com → New repository → name it bca-tracker → Public → Create. Don't add any default files.

6

Edit index.html Before Uploading

Open index.html in any text editor (Notepad, TextEdit). Find the 3 lines near the top that say YOUR_SUPABASE_URL and YOUR_SUPABASE_ANON_KEY and replace them with your actual values. Also change COACH_PIN from '1234' to your chosen coach PIN.

It should look like:

const SUPABASE_URL = 'https://xxxx.supabase.co'; const SUPABASE_KEY = 'eyJhbGci...'; const COACH_PIN = '7391';
7

Upload All 3 Files

In your new GitHub repo click Add file → Upload files. Drag in your edited index.html, manifest.json, and sw.js. Commit.

8

Enable GitHub Pages

In your repo go to Settings → Pages → Source: Deploy from branch → main → / (root) → Save. Wait 2 minutes. Your app URL will be: yourusername.github.io/bca-tracker

9

Share the Link

Send that URL to every athlete. On iPhone: open in Safari → Share button → Add to Home Screen. On Android: Chrome menu → Add to Home Screen. It installs as an app icon.

✓ YOU'RE LIVE

Every athlete creates their profile on first open. All data syncs to your Supabase database. Access the coach dashboard from any athlete's phone via More → Coach Dashboard.
